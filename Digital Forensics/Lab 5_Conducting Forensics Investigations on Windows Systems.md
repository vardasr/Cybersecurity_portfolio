# Overview
The setting of this lab is an ongoing investigation into the activities of Beverly Gates, the HR manager at Intricate Solutions, Inc. Senior leadership has reason to believe that Beverly is involved in a sophisticated drug trafficking operation with ties to the Russian mafia and has recently brought contacted the authorities to investigate the matter further. As a forensic analyst assigned to the case, I have been given a drive image taken from Beverly's work laptop and tasked with reviewing its contents for forensic evidence of suspicious activity.
## Methods
* (Section 1 Part 1) I will use a variety of different tools to gather forensically relevant information about a Windows system.
* (Section 1 Part 2) I will open the Registry Editor and get acquainted with the organizational structure of an Windows Operating System.
* (Section 2 Part 1) I will use Paraben's E3 to create a new case file and import a Windows drive image. I will then use E3's Content Analysis feature to analyze and sort the files on the drive image into different categories that will streamline my search for evidence.
* (Section 2 Part 2) I will analyze the Windows drive file system, and files to discover any potentially incriminating evidence against Beverly Gates.
* (Section 3 Part 1) I will used the Advance Search results to locate a suspicious file that contains suspicious addresses.
* (Section 3 Part 2) I will then go back into the Registry and look for additional evidence related to Tor.
## Summary
In this lab I had many important concepts to learn. First I will use several Windows utilities to perform live analysis on a Windows Server 2019 system. I will then explore one of the most evidence-rich areas of the Windows operating system - Windows Registry. I will finally  shift my attention to a Windows drive image and conduct forensics analysis using Paraben's E3.
